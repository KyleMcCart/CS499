Users can utilize the application to enter user stories and acceptance criteria; to enter sprint information; to choose priority levels; and to track each story through agile workflow stages (to do/in Progress/Done). The artifact represents a practical project management tool that supports agile planning and sprint organization. I selected this artifact because it directly connects to software design & engineering. Agile project management is widely used in software development teams so this project allowed me to demonstrate how an application can support planning/organization and tracking of tasks. I also selected this artifact because it was developed as a basic task tracker with clear opportunities for improvement. Therefore, developing it further allowed me to show growth in front end development, application logic, ui design & data handling. The enhanced version of the Agile Task Tracker displays several important skills in software development. I improved the project by adding form validation so users cannot submit incomplete user stories. I also added editable and deletable task cards, status tracking, priority labels, sprint Progress tracking, and local storage so the information remains saved even after refreshing the page. These updates made the application more useful and professional because now users are able to manage their tasks, update their Progress, and keep their work stored within the browser. These improvements have helped me make Progress toward multiple Computer Science program outcomes. The project demonstrates my ability to develop a computing solution that solves a clear problem using software engineering practices. Additionally, the project supports professional communication as the application presents information in a clean and organized way for users. Finally, the use of JavaScript, localstorage, form validation, and structured task data demonstrates my ability to utilize computing tools and techniques to create functional solutions that deliver value. As i improved this artifact i realized just how important thinking about the experience of the user is not only considering the code. Initially, the project primarily allowed users to input task information. Following the improvement process, it became much more complete agile board including task movement, Progress tracking, editing, deleting, and saving data. This has provided me with better understanding on how applications manage data and update pages based upon actions taken by users. Another challenge i faced was creating a design that would appear professional yet remain simple and easy to use. I improved the layout with columns, task cards, priority colors, responsive styling to create a more authentic feel like a real agile planning tool. Ultimately, this enhancement has provided me with the opportunity to show growth in software design & engineering. I transformed the artifact from a simple form to input tasks into a fully functioning Agile Task Tracker supporting sprint planning & task management. Thus making it a strong addition to my portfolio due to its representation of my capability to enhance existing projects using software development best practices and providing additional functionality for users.

Enhancement Two: Algorithms and Data Structures

For my algorithms and data structures enhancement, I selected the DriverPass Scheduler artifact. The DriverPass Scheduler is a Python-based scheduling program created to manage students, instructors, and driving appointments for a driving school system. The original version of the program used basic dictionaries and lists to store students, instructors, and appointment information. Users could add appointments, view appointments, search for appointments by student name, and view available instructor time slots. I selected this artifact because it directly connects to algorithms and data structures. A scheduling program depends on organized data, search logic, validation, and conflict checking. The original version had useful basic functionality, but it needed stronger logic to better represent a real-world scheduling system. To enhance the artifact, I added date and time validation, double-booking protection for students and instructors, appointment status tracking, appointment cancellations, instructor-based appointment searches, and student progress reports. These improvements turned the program from a simple scheduling tool into a more complete appointment management system. The enhanced version demonstrates my ability to use dictionaries, lists, functions, validation logic, and scheduling conflict checks to manage data and prevent errors. This enhancement improved the artifact by making it more reliable and practical for users. Instead of only storing appointment information, the program now checks whether an appointment can actually be scheduled before allowing it. For example, the system can prevent a student or instructor from being booked for two appointments at the same time. This required me to think carefully about how the program stores, searches, and compares appointment data. One challenge I faced during this enhancement was making sure the scheduling logic worked correctly across different user actions. I had to verify that appointments could be added, searched, canceled, updated, and checked for conflicts without breaking the program’s flow. This helped me better understand the importance of testing, validation, and logical program design. Through this artifact, I demonstrated growth in algorithms and data structures by improving how the program organizes and processes data. The DriverPass Scheduler shows my ability to apply structured data, search logic, and validation techniques to solve a practical scheduling problem. Overall, this enhancement supports my ePortfolio by showing that I can take an existing program and improve it into a more complete and useful computing solution.

Enhancement Three: Databases

The artifact i selected for the databases category is the "DriverPass" database application. The artifact is a Python program utilizing SQLite to store and manage information for a driving school system. The application contains records for students, instructors, vehicles, and driving lessons. The original version was developed during earlier coursework as part of the driver pass design & scheduling project. I enhanced the artifact in CS 499 for milestone four to better illustrate professional software development practices, data validation, security awareness, and database design. Why did i choose this artifact? I chose this artifact because it directly relates to database skills. The project shows how an application can use a database to organize real business information and support decision making through reporting and scheduling. Therefore, the artifact is more than a basic program and represents how software can support an organization's operations and decision making. The enhanced artifact demonstrates several skills related to databases. First, i improved the structure of the database by utilizing separate tables for each entity (students, instructors, vehicles, lessons, audit logs). Second, i strengthened the design using primary keys, foreign keys, unique constraints, check constraints and indexes. Third, these enhancements provide the benefits of protecting data quality and providing a reliable database. Fourth, the lesson table now connects students, instructors and vehicles together which provides a better representation of the real world DriverPass scheduling process. In addition to the above enhancements, the artifact has been improved through stronger validation and error handling. The enhanced version checks IDs, email addresses, dates, times, lesson statuses and package types before saving any data. The enhanced version also uses parameterized queries when interacting with the database to reduce the risk of unsafe input into the database. Finally, the program has been enhanced to include an audit log that captures important changes made to the data along with a student progress report that utilizes sql aggregation to summarize completed, scheduled and cancelled lessons. How does the artifact relate to course outcomes? The enhancements to this artifact support many of the Computer Science program outcomes. They support the outcome of implementing computing solutions that deliver value because the project used Python and SQLite to develop a practical database application. They also support the outcome of software design and database practice because i improved the schema relationships, validation and reporting features of the project. Additionally, the enhancements connect to the security mindset outcome. I utilized foreign key enforcement, input validation, parameterized queries, and database constraints to help protect the accuracy and privacy of the data. Although this artifact will not fully demonstrate every course outcome alone, it will provide strong evidence of database competency, software engineering competency communication and secure design thinking. My outcome coverage plan is still the same; however, i now believe this artifact is particularly useful for demonstrating database design and security awareness. What have i learned about enhancements? Developing this artifact demonstrated to me why developing a well-planned database prior to writing or modifying code is so critical. In the initial version of the artifact, although it could be used to schedule lessons, the database design was much less complex. However as i developed the artifact further, i began to develop a greater understanding of how each table should relate to One another and how they can be used to ensure valid and non-duplicate data are saved. Developing a planned database design is not simply a method of saving data; it provides both protection against bad data being stored as well as methods for using the data. A challenge that i encountered during this development was determining how to preclude scheduling conflicts. Since there may be cases where a student, instructor or vehicle cannot be scheduled for two active lessons at the same date/time in a driving school environment; therefore, when developing this solution i utilized database constraints and indexing rather than solely relying upon programming logic. This made the application more dependable since the database enforces its own rules. An additional challenge i encountered was finding a balance between providing simplicity for users as well as presenting high-quality results. I desired the project to continue to be simple to operate/understand yet present measurable improvements over previous versions. I resolved this issue by continuing to utilize a user-friendly menu-based Python interface while improving the database design/validation/reporting within the project. This process provided insight into how even small applications can be significantly improved via database design improvement and adherence to good coding practices.
